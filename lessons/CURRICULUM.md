# What This Class Is Actually About

You will probably never hand-write most of your code. The AI writes faster than you ever will, and it's not close. That doesn't make you obsolete — it changes what the scarce skill is.

The scarce skill is **taste and judgment**: knowing whether what the AI wrote is any good, what's missing, what doesn't belong, and what to do about it. The AI supplies infinite *competence*; you supply the *opinion*. Every daily lesson in this folder is secretly training one of the nine judgment skills below. When you finish a lesson, you should be able to name which one.

## The Nine Skills

**1. Where things live.** Style belongs in CSS, behavior in JS, structure in HTML — every fact gets exactly one home. AI writes *locally correct* code that puts things in the wrong place, because the snippet works even when the architecture suffers. *Ask: does this belong here?*

**2. Spotting duplication.** Regenerating the same 120 lines costs the AI nothing; it costs you four places to fix every bug. The model will never flag its own repetition — noticing it is your job. *Ask: have I seen this code before?*

**3. Deletion and restraint.** AI only ever adds. Subtraction is pure human judgment, and it's the skill no one practices. A feature that doesn't earn its complexity is a liability with good lighting. *Ask: what would I remove?*

**4. Reading code you didn't write.** This replaces typing as the core literacy. If you can't trace what happens when the button is clicked, you don't own the code — you're just hosting it. *Ask: could I explain this line to someone else?*

**5. Defining "done" and proving it.** Generated code looks finished whether or not it works. Test every state — especially failure. Measure instead of assuming. Break it on purpose and see what catches it. *Ask: how do I know this works?*

**6. Git as the safety net.** Small commits, honest messages, fearless reverts. Aggressive AI use is only safe when every experiment is cheap to undo. A revert isn't a failure; it's the system working. *Ask: if this goes wrong, how do I get back?*

**7. Writing intent before code.** The new programming is specification: saying precisely what you want, with constraints and acceptance criteria, *before* anything is generated. Vague prompt in, plausible garbage out. *Ask: did I decide what I want, or did the AI decide for me?*

**8. Steering the agent.** AGENTS.md is your steering wheel, not homework. Encoding your conventions, banning your pet peeves, and updating the rules as the project evolves — that's you governing your AI instead of the reverse. *Ask: should this decision become a rule?*

**9. The quality the AI skips.** Accessibility, error handling, performance, contrast — the dimensions where failure lives but nothing sparkles. Models invest where it's visually impressive unless you demand otherwise. *Ask: what did it NOT do?*

## How to use this page

When you review anything the AI wrote — and reviewing is now most of the job — run the nine questions. At first it's a checklist. Eventually it's a reflex. That reflex is the thing this class exists to build, because it's the one thing the AI can't generate for you.
