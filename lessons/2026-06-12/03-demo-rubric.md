# Lesson 3: Demo, Score, and Explain It

**Trains:** Skill #8 - Steering the agent, and Skill #9 - Review judgment.

**Time estimate:** 45-60 minutes

## The demo

You get 90 seconds.

Show:

1. the original benchmark result;
2. the feature you built;
3. the file you changed;
4. the test you ran;
5. one thing you would improve next.

## Scoring

Total: 10 points.

### 2 points: Real benchmark

- 1 point for testing all three demos.
- 1 point for honest notes about performance or failures.

### 2 points: Scope control

- 1 point for choosing one project.
- 1 point for keeping the change inside the planned files.

### 2 points: Working behavior

- 1 point for a feature that works with real camera input.
- 1 point for clear failure behavior when camera or model loading fails.

### 2 points: Code ownership

- 1 point for explaining the changed function or event flow.
- 1 point for naming the MediaPipe output data your feature uses.

### 2 points: Taste and clarity

- 1 point for readable UI.
- 1 point for avoiding unnecessary complexity.

## Review questions

Be ready to answer:

- What file did you start in, and why?
- What data comes from MediaPipe?
- What state did your code add?
- What runs once, and what runs every frame?
- What is the weakest part of your solution?
- What would you delete if you had to simplify it?

## Done when

- [ ] Your benchmark table is complete.
- [ ] Your project works from `localhost`.
- [ ] You have at least one commit with a specific message.
- [ ] You can demo without reading from the code the whole time.
